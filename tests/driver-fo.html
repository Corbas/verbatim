Description

The verbatim processor entry point. In order to make this as flexible as possible each component (opening tag, content, close tag) is processed by applying templates to the same element in a different mode.

All of the parameters have default values defined as stylesheet parameters in order to allow overriding at both the application and the node rendering points. Parameters are all tunneled through to lower level templates.

Some tunneled parameters are calculated in this template and passed on:

  • tab-out — the string that literal tabs are to replaced with. Generated from the tab-width and indent-string parameters.
  • verbatim-root — the top level node being processed by the verbatim processor.

Parameters

max-indent

Set max-indent to limit the level to which the stylesheet will indent nested elements. Defaults to the value of the verbatim:max-indent-default stylesheet parameter.

max-depth

Set max-depth to override the depth to which this stylesheet will traverse the input document before replacing the child nodes of the current node with ellipses. Defaults to the value of the verbatim:max-depth-default stylsheet parameter.

replace-entities

Controls whether or not the base XML entities are escaped in element content or not. Defaults to the value of the verbatim:replace-entities-default stylesheet parameter.

indent-elements

Controls whether or not elements are indented as nesting increases. Defaults to the value of the verbatim:indent-elements-default stylesheet parameter.

suppress-ns-declarations

If set to true, suppress-ns-declarations causes all namespace declarations to be omitted from the output. If not set, the value of the verbatim:suppress-ns-declarations-default stylesheet parameter is used.

suppressed-namespaces

Set a sequence of URIs in the suppressed-namespaces parameter in order to always skip declarations for those namespaces. This allows sample code to be placed in a namespace and output without one. Defaults to the value of the verbatim:suppressed-namespaces-default stylesheet parameter.

limit-text

If set to true, the amount of text output as part of a text node will be limited to the number of words defined by the max-words parameter. All of the text after the limit will be replaced by the value of the ellipsis-string parameter. Defaults to the value of the verbatim:limit-text-default stylesheet parameter.

ellipsis-string

The character to be used to replace trailing text longer than the max-words parameter and when the maximum depth has been exceeded. Defaults to the value of the verbatim:ellipsis-string-default stylesheet parameter.

indent-string

The string to be used when building up indent strings. Defaults to the value of the the verbatim:indent-string-default stylesheet parameter.

tab-width

Sets value of the tab-width parameter, used for tab expansions. Defines the number of spaces that will be used to replace a tab character. Defaults to the value of the tab-width-default stylesheet parameter. Generates the verbatim:tab-out tunnel parameter available to all further templates.

max-words

Defines the maximum number of words to be output in any given text node before truncation occurs. If the limit-text parameter is set to false, this has no effect. Defaults to the value of the verbatim:max-words-default stylesheet parameter.

keep-words

Defines the number of words retained when text truncation occurs. keep-words words will be retained each side of the ellipsis when truncation occurs. Defaults to the value of the verbatim:keep-words-default stylesheet parameter.

normalize-text

If set to true() then all text is normalized before output. Defaults to the value of the verbatim:normalize-text-default stylesheet parameter.
